# GameHub AI 自动化工作流详细配置
================================================

本文件详细说明了如何设置和运行GameHub的完整AI自动化内容发布系统

1. AI攻略自动生成和分发工作流
2. 兑换码自动获取和验证
3. 游戏内容更新检测
4. 社交媒体自动发布

## 工作流概述

### 工作流1: 每日内容自动检测与更新

触发: 每天 UTC 00:00

这个工作流负责检查游戏更新和最新内容。

任务:
1. 检查热门游戏更新
2. 检测新发布游戏
3. 更新游戏信息
4. 生成新的攻略内容
5. 清理过期内容

触发频率: 每日一次

环境变量要求:
- ANTHROPIC_API_KEY: Claude API密钥
- DATABASE_URL: 数据库连接字符串
- REDIS_URL: Redis缓存连接

### 工作流2: 兑换码自动监控和更新

触发: 每小时一次

这个工作流自动监控和更新兑换码，包括:

监控源:
1. Reddit相关游戏论坛
2. Discord游戏社区
3. 官方游戏官网和社交媒体

功能:
1. 扫描新兑换码
2. 验证代码验证
3. 自动添加到数据库
4. 标记过期代码

### 工作流3: 游戏攻略自动生成

触发: 有新内容需求时

这个工作流使用AI生成高质量的游戏攻略。

输入:
1. 游戏名称和主题
2. 目标难度级别
3. 内容类型

输出:
1. 结构化的Markdown内容
2. 攻略文章
3. Tier List排名
4. 指南和教程

### 工作流4: 社交媒体自动发布

触发: 有新内容发布时

这个工作流自动将内容分享到社交媒体。

平台:
- Twitter/X
- Reddit
- Discord

## 设置指南

### 本地开发测试

在本地开发环境，首先确保所有自动化工作流，可以执行以下测试步骤:

1. 确保数据库有内容
2. 配置环境变量
3. 运行种子数据

### 生产环境部署准备

在部署到生产环境之前，确保:
1. 所有环境变量已配置
2. 数据库已正确配置
3. 错误报告和日志设置
4. 错误监控和警报

## 常见问题

### 内容生成失败
- 检查 API Key
- 检查配额限制
- 查看日志详情
