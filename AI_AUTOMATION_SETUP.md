# AI 自动化系统配置指南

## 概述

GameHub 的 AI 自动化系统基于 Claude Opus 4 构建，提供以下功能：

- 自动生成游戏攻略
- AI 驱动的内容创作
- 兑换码自动检测和更新
- 自动化发布和更新文章和内容

## 前置要求

### 必要的 API 密钥和配置

以下是设置 AI 自动化系统所必需的 API 密钥和配置项：

### 环境变量配置（所有必需的密钥）

确保以下环境变量在您的 .env.local 或部署环境中正确配置：

| 环境变量名 | 描述 | 获取位置/方式 | 是否必需 |
|------------|------|---------------|----------|
| ANTHROPIC_API_KEY | Claude API 密钥，用于 AI 内容生成 | https://console.anthropic.com/ | 是（核心功能必需） |
| DATABASE_URL | 数据库连接 URL | 本地为 SQLite，部署为 Prisma 支持的数据库 | 是 |
| REDIS_URL / UPSTASH_REDIS_REST_URL 和 UPSTASH_REDIS_REST_TOKEN | Redis 缓存和数据存储配置 | https://upstash.com/ | 推荐用于生产环境（高性能和速率限制管理） |
| INTERNAL_API_SECRET | 内部 API 安全密钥 | 本地生成，确保安全 | 是 |

## 工作流程自动化（使用 n8n 或类似工具）

### 自动化内容生成工作流程建议

以下是推荐的自动化工作流程设置：

1. **游戏更新自动检测和内容更新触发**
   - 检测游戏官方更新和补丁说明
   - 自动更新相关的攻略和内容

2. **定期内容刷新和更新机制**
   - 定时检查和更新所有攻略和内容，保证内容及时性和有效性

## 设置步骤

### 1. 配置环境变量

确保所有必要的 API 密钥和配置项都已在您的部署环境中正确配置。

### 2. 数据库和缓存初始化与设置

通过 Prisma 进行数据库迁移和种子数据设置：

```bash
npm run db:push
npm run db:seed
```

### 3. 测试 API 端点是否工作正常

检查所有 API 端点是否正常响应和工作。启动应用，检查：

```bash
npm run dev
```

然后在浏览器中访问 http://localhost:3000，检查网站是否正常加载。
