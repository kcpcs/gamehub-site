## 📋 执行前必读

### 核心原则
1. **前后端严格分离**: 前端绝不修改后端代码，后端绝不修改前端代码
2. **大模型分工优先**: 按 MODEL_ASSIGNMENT.md 规定的模型分工执行
3. **进度必记录**: 每个任务执行前更新进度追踪，执行后记录结果
4. **问题速上报**: 发现影响项目运行的问题立即暂停并上报

### 大模型分工表 (强制执行)

| 模型 | 负责任务类型 | 优先级 | 降级策略 |
|------|-------------|--------|---------|
| **Claude Opus 4** | 系统架构、核心业务、跨模块协调、Phase 2-5 主要任务 | 🔴 主力 | Opus → Sonnet → Haiku |
| **Claude Sonnet 4** | 单页面优化、API完善、中等复杂度任务 | 🟡 辅助 | Sonnet → Haiku |
| **Claude Haiku** | 简单任务、文档生成、批量处理 | 🟢 快速 | 无降级 |
| **GPT-4o** | 多模态、视觉设计 | 🟢 补充 | 降级到 Sonnet |

### 工具与技能分配

| 工具/技能 | 适用模型 | 使用场景 |
|----------|---------|---------|
| **TodoWrite** | 所有模型 | 任何任务开始前 |
| **Read** | 所有模型 | 查看代码时 |
| **SearchCodebase** | Opus/Sonnet | 查找代码模式时 |
| **RunCommand** | 所有模型 | 需要运行 npm/其他命令时 |
| **Edit** | 所有模型 | 修改现有文件时 |
| **Write** | 所有模型 | 创建新文件时 |
| **Glob** | Haiku/Sonnet | 批量查找文件时 |
| **GetDiagnostics** | 所有模型 | 完成代码后验证时 |
| **search** | Claude Opus 4 | 高难度跨模块搜索 |
